**LangChain** 是目前最流行的 LLM 应用开发框架（GitHub ⭐ 142k+）。

## 二、LangChain 架构概览

LangChain 整体数据流（LCEL 管道，按源码实际执行路径）：

```
用户输入（str / dict / HumanMessage）
   ↓ ① ChatPromptTemplate.from_messages()   格式化为 List[BaseMessage]
   ↓ ② ChatOpenAI / init_chat_model()       调用 LLM，返回 AIMessage
   ↓ ③ StrOutputParser / PydanticParser     解析输出为 str / Pydantic 对象
输出（str / dict / Pydantic）

— RAG 扩展路径 —
文档 → TextSplitter → Embeddings → VectorStore → Retriever
   ↓ ④ retriever.invoke(query)              返回 List[Document]
   ↓ 注入 ② 的 context 字段                组合到 Prompt 中

— Agent 扩展路径 —
   ↓ ⑤ @tool 装饰器定义工具函数
   ↓ ⑥ create_agent()（v1.x 推荐）        LLM 自动决策调用哪个工具
   ↓    create_tool_calling_agent()         旧式写法，需 langchain-classic
   ↓ ⑦ agent.invoke({"messages": [...]})  循环执行工具调用，返回最终答案
```

### 2.1 组件一览（均经官方源码验证）

| 类别 | 官方模块路径 | 类/函数名 |
|------|------------|----------|
| 模型统一初始化 | `langchain` | `init_chat_model()` |
| Chat Model | `langchain_openai` | `ChatOpenAI` |
| Prompt 模板 | `langchain_core.prompts` | `ChatPromptTemplate` / `PromptTemplate` |
| 输出解析器 | `langchain_core.output_parsers` | `StrOutputParser` / `JsonOutputParser` |
| 结构化输出 | `pydantic` | `BaseModel` + `model.with_structured_output()` |
| 文档加载 | `langchain_community.document_loaders` | `TextLoader` / `WebBaseLoader` / `PyPDFLoader` |
| 文本分块 | `langchain_text_splitters` | `RecursiveCharacterTextSplitter` |
| 向量数据库 | `langchain_chroma` | `Chroma` |
| 嵌入模型 | `langchain_openai` | `OpenAIEmbeddings` |
| 检索器 | `langchain_core.vectorstores` | `VectorStore.as_retriever()` |
| LCEL 管道 | `langchain_core.runnables` | `RunnablePassthrough` / `RunnableLambda` |

## 三、Chat Model（模型接口）

### 3.1 使用 init_chat_model 统一初始化

```python
# ============================================================
# init_chat_model() — 跨 Provider 统一模型初始化接口
# 格式："provider:model_name"，无需更改其余代码即可切换模型
# ============================================================
from langchain.chat_models import init_chat_model  # init_chat_model: 统一模型初始化函数

# init_chat_model(model, **kwargs) -> BaseChatModel
#   model      : str  — "provider:model_name" 格式字符串
#                       provider 可为 openai / anthropic / google_genai / ollama 等
#   temperature: float — 采样温度，0.0 输出最确定，1.0 最随机
#                        默认值由底层 provider 决定（OpenAI 默认 1.0，不指定时沿用 provider 默认值）
#   返回值      : BaseChatModel — 实现 Runnable 接口的模型对象
llm = init_chat_model(
    "openai:gpt-4o-mini",  # str: 模型标识，格式为 "provider:model_name"
    temperature=0.0,        # float: 采样温度，设为 0.0 保证输出稳定可复现
)

# model.invoke(input) -> AIMessage
#   input  : str / List[BaseMessage] / dict — 输入内容
#   返回值  : AIMessage — 包含 .content(str) 和 .response_metadata(dict) 字段
response = llm.invoke("你好，请用一句话介绍 LangChain 的核心价值。")  # AIMessage 对象
print(response.content)         # str: LLM 实际生成的文本内容
print(response.response_metadata)  # dict: Token 使用量、模型名称等元数据
```

### 3.2 直接使用 ChatOpenAI

```python
# ============================================================
# ChatOpenAI — OpenAI Chat Model 的直接封装类
# 支持 OpenAI API 及所有兼容 /v1/chat/completions 端点的服务
# ============================================================
from langchain_openai import ChatOpenAI  # ChatOpenAI: OpenAI 聊天模型封装类

# ChatOpenAI(**kwargs) -> ChatOpenAI
#   model      : str   — 模型名称（gpt-4o / gpt-4o-mini / gpt-3.5-turbo 等）
#   temperature: float — 采样温度，默认由 OpenAI 决定（通常 1.0）
#   base_url   : str   — 自定义 API 端点（用于中转站/私有部署，可选）
#   api_key    : str   — 覆盖环境变量 OPENAI_API_KEY（不推荐明文写入代码）
#   max_tokens : int   — 单次生成的最大 Token 数，None 表示不限制
#   注意：streaming 参数已废弃；流式/非流式由调用方式决定（.invoke() vs .stream()）
chat_model = ChatOpenAI(
    model="gpt-4o-mini",     # str: 模型名称，gpt-4o-mini 性价比高
    temperature=0.7,          # float: 采样温度，适度随机
    base_url=OPENAI_BASE_URL, # str: API 地址，支持中转站
    # 注意：不需要显式设置 streaming 参数。
    # 流式/非流式由调用方式决定：.invoke() 返回完整结果，.stream() 返回迭代器
)

# 直接传入 str，ChatOpenAI 内部自动封装为 HumanMessage
result = chat_model.invoke("请解释什么是 LCEL？")  # AIMessage 对象
print(result.content)  # str: 模型生成的回答文本
```

### 3.3 流式输出

```python
# ============================================================
# 流式输出 — stream() 方法逐 Token 实时打印，适合交互式场景
# ============================================================
from langchain_openai import ChatOpenAI  # ChatOpenAI: OpenAI 聊天模型封装类

stream_model = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)  # ChatOpenAI 实例

# model.stream(input) -> Iterator[AIMessageChunk]
#   input    : str / List[BaseMessage] — 输入内容
#   返回值    : Iterator[AIMessageChunk] — 逐步产生的文本片段迭代器
# 每个 chunk.content 是 str 类型的局部文本（一个或多个 Token）
for chunk in stream_model.stream("写一首关于 AI 的五言绝句"):  # AIMessageChunk 对象
    print(chunk.content, end="", flush=True)  # str: 实时打印每个 Token 片段，不换行
print()  # 流式结束后换行
```

### 3.4 批量调用

```python
# ============================================================
# 批量调用 — batch() 并行处理多个输入，比循环 invoke() 更高效
# ============================================================
from langchain_openai import ChatOpenAI  # ChatOpenAI: OpenAI 聊天模型封装类

batch_model = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)  # ChatOpenAI 实例

# model.batch(inputs) -> List[AIMessage]
#   inputs  : List[str | List[BaseMessage]] — 多个输入的列表
#   返回值   : List[AIMessage] — 与输入一一对应的响应列表
questions = [             # List[str]: 批量问题列表
    "Python 是什么语言？",
    "RAG 的全称是什么？",
    "向量数据库有什么用？",
]
responses = batch_model.batch(questions)  # List[AIMessage]: 批量响应结果
for q, r in zip(questions, responses):   # q: str 问题，r: AIMessage 响应
    print(f"Q: {q}\nA: {r.content}\n")   # 打印问答对
```

## 四、Prompt Template（提示词模板）

### 4.1 ChatPromptTemplate

```python
# ============================================================
# ChatPromptTemplate — 多角色对话提示词模板
# 支持 system / human / ai / placeholder 四种消息角色
# ============================================================
from langchain_core.prompts import ChatPromptTemplate  # ChatPromptTemplate: 对话提示词模板类
from langchain_openai import ChatOpenAI                # ChatOpenAI: OpenAI 聊天模型类

# ChatPromptTemplate.from_messages(messages) -> ChatPromptTemplate
#   messages: List[Tuple[str, str]] — (角色, 模板字符串) 的列表
#             角色可选值：
#               "system"      — 系统级指令，设定 AI 的行为和身份
#               "human"       — 用户消息，支持 {变量名} 占位符
#               "ai"          — 预设的 AI 回复（Few-shot 示例）
#               "placeholder" — 动态插入完整消息列表（用于 Memory/Agent）
#   返回值  : ChatPromptTemplate — 可调用的模板对象
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一位专业的 {domain} 领域专家，用简洁易懂的语言回答问题。"),
    # {domain} : str — 动态注入的领域名称
    ("human",  "{question}"),
    # {question}: str — 动态注入的用户问题
])

# prompt.format_messages(**kwargs) -> List[BaseMessage]
#   返回值: List[BaseMessage] — 格式化后的消息列表，可直接传给 LLM
formatted = prompt.format_messages(
    domain="人工智能",    # str: 填充 system 消息中的 {domain} 占位符
    question="什么是 Transformer 架构？"  # str: 填充 human 消息中的 {question} 占位符
)
for msg in formatted:              # msg: BaseMessage 对象（SystemMessage / HumanMessage）
    print(f"{msg.type}: {msg.content}")  # str: 消息角色类型 + 消息内容

# prompt.invoke(input_dict) -> ChatPromptValue
#   等价于 format_messages，返回 ChatPromptValue（可直接作为链的输入）
prompt_value = prompt.invoke({"domain": "数据科学", "question": "解释过拟合"})
print(prompt_value.to_messages())  # List[BaseMessage]: 消息列表
```

### 4.2 Few-Shot 提示词（少样本示例）

```python
# ============================================================
# FewShotChatMessagePromptTemplate — Few-Shot 提示词
# 通过提供示例，引导模型输出符合期望格式的回答
# ============================================================
from langchain_core.prompts import (
    ChatPromptTemplate,            # ChatPromptTemplate: 多角色对话模板
    FewShotChatMessagePromptTemplate,  # FewShotChatMessagePromptTemplate: Few-Shot 模板
)

# 示例数据 — List[Dict[str, str]]，每条包含 input 和 output 键
examples = [  # List[Dict]: 示例列表，每条为一个输入/输出对
    {"input": "苹果",  "output": "水果，红色，甜"},  # Dict: 单条 Few-Shot 示例
    {"input": "香蕉",  "output": "水果，黄色，软"},  # Dict: 单条 Few-Shot 示例
    {"input": "西红柿", "output": "蔬果，红色，酸"},  # Dict: 单条 Few-Shot 示例
]

# 定义单条示例的模板格式 — 输入为 human 角色，输出为 ai 角色
example_prompt = ChatPromptTemplate.from_messages([
    ("human", "{input}"),   # str: 示例中用户输入的模板
    ("ai",    "{output}"),  # str: 示例中 AI 输出的模板
])

# FewShotChatMessagePromptTemplate(examples, example_prompt) -> FewShotChatMessagePromptTemplate
#   examples      : List[Dict] — 示例数据列表
#   example_prompt: ChatPromptTemplate — 单条示例的格式模板
few_shot_prompt = FewShotChatMessagePromptTemplate(
    examples=examples,          # List[Dict]: Few-Shot 示例数据
    example_prompt=example_prompt,  # ChatPromptTemplate: 示例格式模板
)

# 拼接完整提示词
final_prompt = ChatPromptTemplate.from_messages([
    ("system", "请仿照示例格式，描述给定事物的属性（类别、颜色、特点）。"),
    few_shot_prompt,    # FewShotChatMessagePromptTemplate: 展开为多条 human/ai 消息
    ("human", "{input}"),  # str: 真实用户输入的占位符
])

# invoke 测试效果
print(final_prompt.format_messages(input="草莓"))  # List[BaseMessage]: 完整消息列表
```

## 五、Output Parser（输出解析器）

### 5.1 StrOutputParser — 提取纯文本

```python
# ============================================================
# StrOutputParser — 最常用的输出解析器
# 将 AIMessage 对象转换为纯字符串（提取 .content 字段）
# ============================================================
from langchain_core.output_parsers import StrOutputParser  # StrOutputParser: 字符串输出解析器
from langchain_core.prompts import ChatPromptTemplate      # ChatPromptTemplate: 对话模板
from langchain_openai import ChatOpenAI                    # ChatOpenAI: OpenAI 模型

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)  # ChatOpenAI: 模型实例
prompt = ChatPromptTemplate.from_messages([                 # ChatPromptTemplate: 提示词模板
    ("human", "{input}")  # str: 用户输入占位符
])
parser = StrOutputParser()  # StrOutputParser: 字符串解析器实例，无需参数

# LCEL 管道：prompt → llm → parser
# `|` 运算符的本质：构建 RunnableSequence，而非立即执行
#   chain = prompt | llm | parser
#   等价于：RunnableSequence([prompt, llm, parser])
#   此行仅"组装"流水线，不触发任何调用
#
# 真正执行发生在 chain.invoke(input) 时，内部自动依次执行：
#   ① prompt.invoke({"input": ...})  → List[BaseMessage]（格式化后的消息列表）
#   ② llm.invoke(List[BaseMessage])  → AIMessage（模型返回的消息对象）
#   ③ parser.invoke(AIMessage)       → str（提取 AIMessage.content 字段）
#
# 每一步的返回值自动作为下一步的输入，无需手动传递
#
# StrOutputParser.invoke(message) -> str
#   输入  : AIMessage — LLM 输出的消息对象
#   返回值 : str — AIMessage.content 字段的字符串值
chain = prompt | llm | parser  # RunnableSequence: LCEL 管道对象，输入 dict，输出 str

result = chain.invoke({"input": "用一句话解释向量数据库"})  # str: 解析后的纯文本
print(result)        # str: 直接可用的文本，不含 AIMessage 包装
print(type(result))  # <class 'str'>
```

### 5.2 JsonOutputParser — 解析 JSON 输出

```python
# ============================================================
# JsonOutputParser — 解析模型输出的 JSON 字符串为 Python 字典
# 适用于需要结构化输出（如列表、键值对）的场景
# ============================================================
from langchain_core.output_parsers import JsonOutputParser  # JsonOutputParser: JSON 解析器
from langchain_core.prompts import ChatPromptTemplate       # ChatPromptTemplate: 对话模板
from langchain_openai import ChatOpenAI                     # ChatOpenAI: 模型实例

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)  # ChatOpenAI: 低温保证 JSON 格式稳定
parser = JsonOutputParser()  # JsonOutputParser: JSON 输出解析器，无需参数

# 提示词中明确要求输出 JSON 格式
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个数据提取助手，请严格以 JSON 格式输出，不要包含 Markdown 代码块。"),
    ("human",  "从以下文本提取信息：{text}\n"
```

## 六、LCEL 链式组合（`|` 管道）

### 6.1 LCEL 核心概念

```python
# ============================================================
# LCEL（LangChain Expression Language）核心说明
# ------------------------------------------------------------
# 所有 LangChain 组件均实现 Runnable 接口，提供以下统一方法：
#   .invoke(input)          → 同步调用，返回单个结果
#   .stream(input)          → 流式调用，返回 Iterator[chunk]
#   .batch(inputs)          → 批量并行调用，返回 List[result]
#   .ainvoke(input)         → 异步调用（async/await）
#   .astream(input)         → 异步流式
#   .abatch(inputs)         → 异步批量
#
# | 运算符将两个 Runnable 串联：left | right
#   等价于：RunnableSequence([left, right])
#   left 的输出直接作为 right 的输入
# ============================================================
from langchain_core.runnables import RunnablePassthrough, RunnableLambda  # 辅助 Runnable 组件
from langchain_core.prompts import ChatPromptTemplate                       # 提示词模板
from langchain_core.output_parsers import StrOutputParser                   # 字符串解析器
from langchain_openai import ChatOpenAI                                     # OpenAI 模型

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)  # ChatOpenAI: 模型实例
parser = StrOutputParser()                                   # StrOutputParser: 字符串解析器

# ---- 基础链：prompt | llm | parser ----
basic_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个翻译助手，将中文翻译成英文。"),
    ("human",  "{text}"),  # str: 待翻译文本的占位符
])
translate_chain = basic_prompt | llm | parser  # RunnableSequence: 翻译链

result = translate_chain.invoke({"text": "人工智能正在改变世界"})  # str: 翻译结果
print(f"翻译结果: {result}")  # str: 英文翻译
```

### 6.2 RunnablePassthrough / RunnableLambda / RunnableParallel — 透传、函数包装与并行执行

```python
# ============================================================
# RunnablePassthrough — 将输入原样传递给下游，不做任何变换
# 常用于 RAG 等需要将原始 query 同时送入多个分支的场景
# ============================================================
from langchain_core.runnables import RunnablePassthrough  # RunnablePassthrough: 透传 Runnable
from langchain_core.prompts import ChatPromptTemplate     # ChatPromptTemplate: 提示词模板
from langchain_core.output_parsers import StrOutputParser # StrOutputParser: 字符串解析器
from langchain_openai import ChatOpenAI                   # ChatOpenAI: 模型实例

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)  # ChatOpenAI: 模型实例
parser = StrOutputParser()                                   # StrOutputParser: 字符串解析器

# RunnablePassthrough 用于构造多分支输入 dict
# 场景：需要将原始 question 传给 Prompt 的同时，也把 context 注入进去
rag_prompt = ChatPromptTemplate.from_messages([
    ("system", "请根据以下上下文回答问题：\n\n{context}"),
    ("human",  "{question}"),  # str: 用户问题占位符
])

# 模拟检索函数 — 实际场景替换为向量检索
def fake_retriever(query: str) -> str:  # query: str 检索词，返回 str 相关文档片段
    """模拟检索器，返回与 query 相关的上下文文档（实际应替换为向量数据库检索）。"""
    return f"[模拟检索结果] 关于 '{query}' 的相关文档内容：LangChain 是一个 LLM 应用框架。"

# RunnableLambda — 将普通 Python 函数包装为 Runnable
# 原因：`|` 管道运算符要求两侧对象均实现 Runnable 接口（具备 .invoke()/.stream()/.batch() 等方法）
# 普通 Python 函数不具备 Runnable 接口，无法直接放入 `|` 管道或 RunnableParallel 的 dict 中
# RunnableLambda(fn) 充当"适配器"：将任意 Python 函数包装成 Runnable 对象
#   包装前：fake_retriever 是普通函数，{"context": fake_retriever} 会报错
#   包装后：RunnableLambda(fake_retriever) 具备 Runnable 接口，可无缝插入管道
from langchain_core.runnables import RunnableLambda  # RunnableLambda: 函数 → Runnable 适配器

# 构建 RAG 式链：
# LCEL 中，管道首位的 dict 会被自动识别为 RunnableParallel（并行执行器）
# 它将同一份输入同时分发给 dict 中的每个值（每个值必须是 Runnable），并行执行后
# 将所有分支的结果按 key 合并成一个新 dict，再整体传给下一个组件
#
# 本例执行流程（输入为 str，如 "LangChain 是什么？"）：
#   ┌─ context  分支：fake_retriever("LangChain 是什么？") → "[模拟检索结果]..."  (str)
#   └─ question 分支：原样透传                            → "LangChain 是什么？" (str)
#                ↓ 两路并行执行完毕后合并
#   {"context": "[模拟检索结果]...", "question": "LangChain 是什么？"}  (dict)
#                ↓ 传给 rag_prompt，填充 {context} 和 {question} 占位符
#
# 这是 RAG 的核心模式：同一份用户输入，同时走"检索"和"透传"两条路，
# 拼合后一起送给模型，使模型能基于检索到的知识回答问题
rag_chain = (
    {
        "context":  RunnableLambda(fake_retriever),  # Runnable: 检索分支，输入 str，输出检索文档 str
        "question": RunnablePassthrough(),            # Runnable: 透传分支，原样输出用户问题 str
    }                                                 # dict 整体被识别为 RunnableParallel，并行执行后合并为 dict
    | rag_prompt  # ChatPromptTemplate: 接收 dict，将 context/question 填入模板，返回 List[BaseMessage]
    | llm         # ChatOpenAI: 接收 List[BaseMessage]，调用模型，返回 AIMessage
    | parser      # StrOutputParser: 提取 AIMessage.content，返回 str
)

answer = rag_chain.invoke("LangChain 是什么？")  # str: 最终回答
print(answer)  # str: 基于检索结果生成的回答
```

### 6.3 多步顺序链

```python
# ============================================================
# 多步顺序链 — 第一步输出作为第二步输入
# 适合需要「先做 A，再基于 A 的结果做 B」的工作流
# ============================================================
from langchain_core.prompts import ChatPromptTemplate     # ChatPromptTemplate: 对话模板
from langchain_core.output_parsers import StrOutputParser  # StrOutputParser: 字符串解析器
from langchain_openai import ChatOpenAI                   # ChatOpenAI: 模型实例

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)  # ChatOpenAI: 模型实例
parser = StrOutputParser()                                   # StrOutputParser: 解析器

# 第一步：根据主题生成文章大纲
outline_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一位内容策划师，擅长为技术文章设计清晰的结构。"),
    ("human",  "请为主题「{topic}」生成一个包含 3 个要点的文章大纲。"),
    # {topic}: str — 文章主题
])
outline_chain = outline_prompt | llm | parser  # Runnable: 大纲生成链，输出 str

# 第二步：根据大纲扩写第一段
expand_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一位技术写作专家，擅长把大纲扩写为流畅的段落。"),
    ("human",  "根据以下大纲，扩写第一个要点：\n\n{outline}"),
    # {outline}: str — 第一步生成的大纲文本
])
expand_chain = expand_prompt | llm | parser  # Runnable: 段落扩写链，输出 str

# 串联两步 — outline_chain 的 str 输出直接作为 expand_chain 的 {outline} 输入
# 注意：expand_chain 需要 dict 输入，故使用 lambda 包装
from langchain_core.runnables import RunnableLambda  # RunnableLambda: 函数 → Runnable

full_chain = (
    outline_chain                                           # Runnable: 第一步，输出 str
    | RunnableLambda(lambda outline: {"outline": outline})  # Runnable: 将 str 包装为 dict
    | expand_chain                                          # Runnable: 第二步，输入 dict，输出 str
)

result = full_chain.invoke({"topic": "LangChain LCEL 的设计哲学"})  # str: 最终扩写段落
print(result)  # str: 基于大纲扩写的文章第一段
```

## 七、对话记忆（Memory / History）

### 7.1 使用 ChatMessageHistory 管理对话历史

```python
# ============================================================
# ChatMessageHistory — 内存对话历史管理器
# 将每轮对话存储为 List[BaseMessage]，实现多轮上下文记忆
# ============================================================
from langchain_community.chat_message_histories import ChatMessageHistory  # 内存历史记录器
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder  # 提示词组件
from langchain_core.output_parsers import StrOutputParser                   # 字符串解析器
from langchain_openai import ChatOpenAI                                     # OpenAI 模型

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)  # ChatOpenAI: 模型实例
parser = StrOutputParser()                                   # StrOutputParser: 字符串解析器

# ChatMessageHistory() -> ChatMessageHistory
#   无参数，创建空的内存历史存储器
#   .add_user_message(content)    : 追加 HumanMessage，content: str
#   .add_ai_message(content)      : 追加 AIMessage，content: str
#   .messages                     : List[BaseMessage] — 所有历史消息
history = ChatMessageHistory()  # ChatMessageHistory: 对话历史容器

# MessagesPlaceholder — 在 ChatPromptTemplate 中动态插入消息列表的占位符
# variable_name: str — 对应 invoke() 时传入的键名
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一位友好的 AI 助手，能记住对话历史并基于上下文回答。"),
    MessagesPlaceholder(variable_name="history"),  # 动态插入历史消息列表
    ("human", "{question}"),  # str: 当前用户问题占位符
])

chain = prompt | llm | parser  # Runnable: 带记忆的对话链

def chat_with_memory(question: str) -> str:
    """
    带记忆的对话函数，自动维护历史消息。

    Args:
        question: str — 用户输入的问题

    Returns:
        str — AI 的回答文本
    """
    # chain.invoke(input_dict) -> str
    #   history  : List[BaseMessage] — 历史消息列表，传给 MessagesPlaceholder
    #   question : str — 当前用户问题
    answer = chain.invoke({
        "history":  history.messages,  # List[BaseMessage]: 当前所有历史消息
        "question": question,          # str: 用户当前问题
    })
    history.add_user_message(question)  # 将用户消息追加到历史，None 无返回值
    history.add_ai_message(answer)      # 将 AI 回答追加到历史，None 无返回值
    return answer  # str: AI 的回答文本

# 多轮对话测试
print(chat_with_memory("我叫小明，今年 23 岁，是一名 AI 工程师。"))  # str: 第一轮回答
print(chat_with_memory("你还记得我叫什么名字吗？"))                   # str: 应能正确回答"小明"
print(chat_with_memory("根据我的情况，推荐一个适合我的 LangChain 学习路径。"))  # str: 个性化建议
print(f"\n共 {len(history.messages)} 条历史消息")  # int: 当前历史消息总数
```

### 7.2 RunnableWithMessageHistory — 自动化历史管理（已废弃）

> **⚠️ 废弃说明**：`RunnableWithMessageHistory` 已于 `langchain_core` **1.3.3** 中标记为 deprecated，
> 并将在 **2.0.0** 中移除。官方建议新项目改用 **LangGraph 内置持久化（Checkpointer）**。
> 现有 v1.x 代码继续可用但会触发 `DeprecationWarning`，入门学习了解原理即可，
> 生产环境请参考 [LangGraph 记忆迁移指南](https://github.com/langchain-ai/langchain/blob/master/docs/docs/versions/migrating_memory/index.mdx)。

```python
# ============================================================
# RunnableWithMessageHistory — 自动注入/更新对话历史的 Runnable 包装器
# ⚠️ deprecated since langchain_core 1.3.3，将于 2.0.0 移除
# 通过 session_id 区分不同用户/会话，适合多用户场景（仍可用，但建议迁移至 LangGraph）
# ============================================================
from langchain_core.runnables.history import RunnableWithMessageHistory  # 历史自动注入包装器
from langchain_community.chat_message_histories import ChatMessageHistory  # 内存历史记录器
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder  # 提示词组件
from langchain_core.output_parsers import StrOutputParser                   # 字符串解析器
from langchain_openai import ChatOpenAI                                     # OpenAI 模型

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)  # ChatOpenAI: 模型实例
parser = StrOutputParser()                                   # StrOutputParser: 字符串解析器

# 用字典模拟多用户会话存储 — Dict[str, ChatMessageHistory]
# 生产环境可替换为 Redis / 数据库等持久化方案
store: dict = {}  # Dict[str, ChatMessageHistory]: 会话 ID → 历史记录的映射

def get_session_history(session_id: str) -> ChatMessageHistory:
    """
    根据会话 ID 获取或创建对应的 ChatMessageHistory。

    Args:
        session_id: str — 用户/会话的唯一标识字符串

    Returns:
        ChatMessageHistory — 对应会话的历史记录对象（不存在时自动创建）
    """
    if session_id not in store:                    # 会话不存在时创建新的历史记录
        store[session_id] = ChatMessageHistory()   # ChatMessageHistory: 新建历史容器
    return store[session_id]  # ChatMessageHistory: 返回对应会话的历史记录

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一位专业的技术顾问。"),
    MessagesPlaceholder(variable_name="chat_history"),  # 注意：键名须与下方 history_messages_key 一致
    ("human", "{question}"),  # str: 用户当前问题占位符
])

base_chain = prompt | llm | parser  # Runnable: 基础对话链

# RunnableWithMessageHistory 包装器
#   runnable             : Runnable — 被包装的基础链
#   get_session_history  : Callable[[str], BaseChatMessageHistory] — 历史获取函数
#   input_messages_key   : str — invoke() 输入中用户消息字段的键名
#   history_messages_key : str — 注入历史时对应 MessagesPlaceholder 的变量名
chain_with_history = RunnableWithMessageHistory(
    base_chain,                              # Runnable: 被包装的基础对话链
    get_session_history,                     # Callable: 历史记录获取函数
    input_messages_key="question",           # str: 用户消息在 input dict 中的键名
    history_messages_key="chat_history",     # str: 与 MessagesPlaceholder 的 variable_name 一致
)

# invoke 时通过 config 传入 session_id
# config["configurable"]["session_id"]: str — 当前会话标识
r1 = chain_with_history.invoke(
    {"question": "我在学习 LangChain，有什么入门建议？"},
    config={"configurable": {"session_id": "user_001"}},  # dict: 配置，指定会话 ID
)
print(r1)  # str: AI 第一轮回答

r2 = chain_with_history.invoke(
    {"question": "能具体说说 LCEL 部分吗？"},
    config={"configurable": {"session_id": "user_001"}},  # str: 相同 session_id，延续上下文
)
print(r2)  # str: AI 基于上下文的第二轮回答（应能联系第一轮的 LangChain 建议）
```

### 8.1 文档加载与分块

```python
# ============================================================
# 文档加载 — Document Loaders
# LangChain 提供 50+ 种文档加载器，统一返回 List[Document]
# Document 对象包含两个字段：
```

```python
# ============================================================
# 文本分块 — RecursiveCharacterTextSplitter
# 将长文档递归切割为小块，保证每块不超过 chunk_size 个字符
# 同时保留 chunk_overlap 字符的重叠，避免语义断裂
# ============================================================
from langchain_text_splitters import RecursiveCharacterTextSplitter  # 递归字符分块器
from langchain_core.documents import Document                          # Document: 文档数据类

# RecursiveCharacterTextSplitter(chunk_size, chunk_overlap, separators) -> TextSplitter
#   chunk_size    : int       — 每个块的最大字符数，建议 200~1000
#   chunk_overlap : int       — 相邻块的重叠字符数，建议 chunk_size 的 10%~20%
#   separators    : List[str] — 分割优先级列表，从左到右依次尝试
#                              先尝试按段落("\n\n")→句子("\n")→句子内(".","。")→字符级("")分割
#   length_function: Callable — 计算文本长度的函数，默认 len（字符数），可替换为 token 计数
splitter = RecursiveCharacterTextSplitter(
    chunk_size=200,          # int: 每块最多 200 个字符（中文约 100 个汉字）
    chunk_overlap=30,        # int: 相邻块重叠 30 个字符，减少语义截断
    separators=["\n\n", "\n", "。", "，", " ", ""],  # List[str]: 分割符优先级
)

# splitter.split_documents(documents) -> List[Document]
#   documents: List[Document] — 原始文档列表
#   返回值   : List[Document] — 分块后的文档列表，每块保留原始 metadata
chunks = splitter.split_documents(raw_docs)  # List[Document]: 分块后的文档列表
print(f"分块数量: {len(chunks)}")             # int: 总分块数
for i, chunk in enumerate(chunks):            # i: int 索引，chunk: Document 文档块
    print(f"\n--- 块 {i+1} ({len(chunk.page_content)} 字符) ---")
    print(chunk.page_content)  # str: 每个块的文本内容
```

### 8.2 构建向量存储与检索

```python
# ============================================================
# 向量化 + 存储 — OpenAIEmbeddings + Chroma
# 将文本块转换为高维向量，存入 Chroma 向量数据库，支持语义检索
# ============================================================
from langchain_openai import OpenAIEmbeddings    # OpenAIEmbeddings: OpenAI 嵌入模型封装
from langchain_chroma import Chroma              # Chroma: Chroma 向量数据库封装

# OpenAIEmbeddings(model) -> OpenAIEmbeddings
#   model   : str — 嵌入模型名称
#             text-embedding-3-small : 维度 1536，性价比高，推荐
#             text-embedding-3-large : 维度 3072，精度更高，成本更高
#             text-embedding-ada-002 : 旧版，已不推荐使用
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",  # str: 嵌入模型，1536 维
    base_url=OPENAI_BASE_URL,        # str: API 端点，支持中转站
)

# Chroma.from_documents(documents, embedding, persist_directory) -> Chroma
#   documents         : List[Document] — 待向量化的文档块列表
#   embedding         : Embeddings — 嵌入模型实例，用于将文本转为向量
#   persist_directory : str — 向量库持久化目录路径（None 则仅内存存储，重启丢失）
vectorstore = Chroma.from_documents(
    documents=chunks,                    # List[Document]: 分块后的文档列表
    embedding=embeddings,                # OpenAIEmbeddings: 嵌入模型
    persist_directory="./chroma_db",     # str: 向量库本地存储目录
)
print(f"向量库已创建，共存入 {len(chunks)} 条向量")
# int: 分块后的文档数量，与实际写入向量库的条目数相同
```

### 8.3 构建完整 RAG 链

```python
# ============================================================
# 完整 RAG 链 — 检索 + 生成的端到端 LCEL 管道
# ============================================================
from langchain_core.prompts import ChatPromptTemplate     # ChatPromptTemplate: 对话模板
from langchain_core.output_parsers import StrOutputParser  # StrOutputParser: 字符串解析器
from langchain_core.runnables import RunnablePassthrough  # RunnablePassthrough: 透传组件
from langchain_openai import ChatOpenAI                   # ChatOpenAI: OpenAI 模型

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)  # ChatOpenAI: 低温保证事实性
parser = StrOutputParser()                                   # StrOutputParser: 字符串解析器

# RAG 提示词模板：明确要求模型「只基于给定上下文回答」
rag_prompt = ChatPromptTemplate.from_messages([
    ("system",
     "你是一位知识问答助手。请严格基于以下检索到的上下文内容回答用户问题，"
     "不要使用上下文以外的知识。如果上下文中没有相关信息，请直接说「我不知道」。\n\n"
     "上下文：\n{context}"),
    # {context}: str — 检索到的文档片段，拼接为字符串
    ("human", "{question}"),  # str: 用户问题占位符
])

def format_docs(docs: list) -> str:
    """
    将 List[Document] 格式化为单个字符串，用于注入 Prompt 的 context 字段。

    Args:
        docs: List[Document] — 检索到的文档块列表

    Returns:
        str — 用双换行符分隔的拼接文本
    """
    return "\n\n".join(doc.page_content for doc in docs)  # str: 所有文档块用换行分隔拼接

# 完整 RAG LCEL 链：
# {"context": retriever | format_docs, "question": RunnablePassthrough()}
#   → RunnableParallel（并行两个分支）
#     context 分支 : 将 question 送入 retriever 检索，再 format_docs 格式化为字符串
#     question 分支: RunnablePassthrough 原样传递 question 字符串
#   → rag_prompt  : 注入 context 和 question，格式化为消息列表
#   → llm         : 调用 OpenAI 模型，返回 AIMessage
#   → parser      : 提取 AIMessage.content 为 str
rag_chain = (
    {
        "context":  retriever | format_docs,  # Runnable: 检索→格式化，输入 str，输出 str
        "question": RunnablePassthrough(),     # Runnable: 透传，输入 str，输出 str
    }
    | rag_prompt   # ChatPromptTemplate: 输入 dict，输出 ChatPromptValue
    | llm          # ChatOpenAI: 输入 ChatPromptValue，输出 AIMessage
    | parser       # StrOutputParser: 输入 AIMessage，输出 str
)

# 测试 RAG 链
test_questions = [     # List[str]: 测试问题列表
    "LangChain 是什么时候创建的？",
    "LCEL 的 | 运算符有什么特点？",
    "LangChain 可以用来做哪些类型的应用？",
]
for q in test_questions:                  # q: str 测试问题
    print(f"\n问题：{q}")
    answer = rag_chain.invoke(q)          # str: RAG 链生成的回答
    print(f"回答：{answer}")
```

### 9.1 定义工具（@tool 装饰器）

```python
# ============================================================
# @tool 装饰器 — 将 Python 函数包装为 LangChain Tool
# Tool 对象包含：名称、描述（从 docstring 提取）、参数 schema（从类型注解生成）
# Agent 通过工具描述自动决策何时调用哪个工具
# ============================================================
from langchain_core.tools import tool  # tool: 将函数装饰为 Tool 的装饰器
import math                            # math: 数学运算标准库
import json                            # json: JSON 处理标准库

@tool
def calculator(expression: str) -> str:
    """
    安全的数学表达式计算器。支持加减乘除、幂运算、平方根等基本运算。

    Args:
        expression: str — 数学表达式字符串，如 "2 + 3 * 4" 或 "sqrt(16)"

    Returns:
        str — 计算结果，失败时返回错误信息
    """
    try:
        # eval 安全限制：只允许访问 math 模块中的函数
        allowed_names = {k: v for k, v in math.__dict__.items() if not k.startswith("_")}
        # Dict[str, Any]: 允许 eval 访问的名称空间（仅 math 模块函数）
        result = eval(expression, {"__builtins__": {}}, allowed_names)
        # eval(expr, globals, locals): 在受限命名空间中执行表达式，返回计算结果
        return str(result)  # str: 将计算结果转换为字符串返回
    except Exception as e:  # Exception: 捕获所有计算异常
        return f"计算错误: {e}"  # str: 返回错误描述，让 Agent 感知失败

@tool
def get_weather(city: str) -> str:
    """
    获取指定城市的天气信息（模拟实现，实际场景替换为真实天气 API）。

    Args:
        city: str — 城市名称，如"北京"、"上海"、"广州"

    Returns:
        str — JSON 格式的天气信息字符串
    """
    # 模拟天气数据 — Dict[str, Dict]: 城市 → 天气信息的映射
    weather_data = {
        "北京": {"温度": "25°C", "天气": "晴天", "湿度": "40%"},
        "上海": {"温度": "28°C", "天气": "多云", "湿度": "65%"},
        "广州": {"温度": "32°C", "天气": "雷阵雨", "湿度": "80%"},
    }
    data = weather_data.get(city, {"错误": f"未找到城市 '{city}' 的天气数据"})
    # Dict: 获取对应城市的天气信息，不存在时返回错误提示
    return json.dumps(data, ensure_ascii=False)  # str: JSON 格式的天气数据字符串

@tool
def search_knowledge(query: str) -> str:
    """
    在 LangChain 知识库中检索相关信息（使用前面构建的 RAG 检索器）。

    Args:
        query: str — 检索查询词

    Returns:
        str — 检索到的相关文档内容
    """
    docs = retriever.invoke(query)       # List[Document]: 检索结果文档列表
    return format_docs(docs)             # str: 格式化后的文档内容字符串

# 打印工具信息
print(f"工具名称: {calculator.name}")   # str: 工具名称，来自函数名
print(f"工具描述: {calculator.description[:80]}...")  # str: 工具描述，来自 docstring
print(f"工具参数: {calculator.args}")   # dict: 工具参数 schema，来自类型注解

tools = [calculator, get_weather, search_knowledge]  # List[Tool]: 工具列表，传给 Agent
```

### 9.2 构建 Tool-Calling Agent

```python
# ============================================================
# create_tool_calling_agent — 构建工具调用 Agent
# 依赖模型的 Function Calling / Tool Calling 能力（GPT-4o 等）
# Agent 在每一步自动决策：回答用户 OR 调用某个工具
# ============================================================
from langchain_classic.agents import create_tool_calling_agent, AgentExecutor
# create_tool_calling_agent / AgentExecutor 在 LangChain v1.x 中已迁至 langchain-classic；
# 新项目推荐使用 langchain.agents.create_agent（见九、3），旧代码迁移时改用此导入。
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder  # 提示词组件
from langchain_openai import ChatOpenAI                                     # OpenAI 模型

# Agent 用的模型必须支持 Tool Calling（gpt-4o / gpt-4o-mini / claude-3 等）
agent_llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)  # ChatOpenAI: 低温保证决策稳定

# Agent 的 Prompt 须包含以下三个占位符：
#   {chat_history}    : MessagesPlaceholder — 对话历史（多轮场景）
#   {input}           : str — 用户当前输入
#   {agent_scratchpad}: MessagesPlaceholder — Agent 思考过程（工具调用记录）
agent_prompt = ChatPromptTemplate.from_messages([
    ("system",
     "你是一位功能强大的 AI 助手，配备了数学计算、天气查询和知识库检索工具。"
     "请根据用户需求，智能选择合适的工具来回答问题。"
     "如果不需要工具，直接回答即可。"),
    MessagesPlaceholder(variable_name="chat_history", optional=True),
    # optional=True: chat_history 不存在时不报错，便于无记忆场景使用
    ("human", "{input}"),                                  # str: 用户当前输入
    MessagesPlaceholder(variable_name="agent_scratchpad"), # Agent 工具调用记录（必须）
])

# create_tool_calling_agent(llm, tools, prompt) -> Runnable（Agent 核心逻辑）
#   llm    : BaseChatModel — 支持 Tool Calling 的模型
#   tools  : List[Tool]   — 工具列表
#   prompt : ChatPromptTemplate — 包含上述三个占位符的提示词模板
#   返回值  : Runnable — Agent 的单步推理逻辑（不含执行循环）
agent = create_tool_calling_agent(
    llm=agent_llm,        # ChatOpenAI: 工具调用模型
    tools=tools,          # List[Tool]: 工具列表（calculator / get_weather / search_knowledge）
    prompt=agent_prompt,  # ChatPromptTemplate: Agent 提示词模板
)

# AgentExecutor — Agent 执行器，负责循环执行「推理 → 工具调用 → 观察 → 推理」直到完成
# AgentExecutor(agent, tools, verbose, max_iterations, handle_parsing_errors)
#   agent            : Runnable — create_tool_calling_agent 返回的 Agent 核心逻辑
#   tools            : List[Tool] — 与创建 Agent 时相同的工具列表
#   verbose          : bool — True 打印每步的推理和工具调用过程，便于调试
#   max_iterations   : int — 最大循环轮次，防止无限循环，默认 15
#   handle_parsing_errors: bool — True 时遇到解析错误自动重试，而非抛出异常
agent_executor = AgentExecutor(
    agent=agent,               # Runnable: Agent 核心推理逻辑
    tools=tools,               # List[Tool]: 可用工具列表
    verbose=True,              # bool: 开启详细日志，打印 Agent 的思考过程
    max_iterations=10,         # int: 最多循环 10 步防止失控
    handle_parsing_errors=True # bool: 解析错误时自动重试
)

# AgentExecutor.invoke(input_dict) -> Dict[str, Any]
#   input_dict: {"input": str} — 用户输入字典
#   返回值    : Dict — {"input": str, "output": str, ...} 包含最终回答
result = agent_executor.invoke({
    "input": "帮我计算 sqrt(144) + 2^8 的结果，然后告诉我北京今天的天气",
    "chat_history": [],  # List: 空历史（首次对话）
})
print(f"\n最终回答: {result['output']}")  # str: Agent 的最终回答
```

### 10.1 使用 Pydantic 定义输出结构

```python
# ============================================================
# with_structured_output() — 强制模型输出符合 Pydantic schema 的 JSON
# 相比 JsonOutputParser，类型更安全，自动验证字段
# ============================================================
from langchain_openai import ChatOpenAI  # ChatOpenAI: OpenAI 模型
from pydantic import BaseModel, Field    # BaseModel: Pydantic 数据模型基类；Field: 字段描述工具
from typing import List, Optional        # List: 列表类型；Optional: 可选类型

# 定义期望的输出结构 — 继承 Pydantic BaseModel
class ArticleAnalysis(BaseModel):
    """
    文章分析结果的结构化输出模型。

    Attributes:
        title      : str — 文章标题
        main_topic : str — 文章主题
        key_points : List[str] — 3~5 个关键要点
        sentiment  : str — 情感倾向（正面/负面/中性）
        word_count : Optional[int] — 估计字数（可选）
    """
    title:      str        = Field(description="文章的标题")
    main_topic: str        = Field(description="文章讨论的核心主题，一句话概括")
    key_points: List[str]  = Field(description="文章的 3~5 个关键要点，每条简洁明了")
    sentiment:  str        = Field(description="文章的情感倾向：正面、负面或中性")
    word_count: Optional[int] = Field(default=None, description="估计的文章字数（可选）")

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)  # ChatOpenAI: 低温保证输出格式稳定

# model.with_structured_output(schema) -> Runnable

#   返回值    : Runnable — 输出自动解析为 schema 指定类型的对象
structured_llm = llm.with_structured_output(ArticleAnalysis)  # Runnable: 结构化输出链

])

# 完整结构化分析链
analyze_chain = analyze_prompt | structured_llm  # Runnable: 输入 dict，输出 ArticleAnalysis

sample_article = """

"""  # str: 待分析的示例文章文本

result = analyze_chain.invoke({"article": sample_article})  # ArticleAnalysis: 结构化输出对象
print(f"标题: {result.title}")         # str: 文章标题
print(f"主题: {result.main_topic}")    # str: 核心主题
print(f"要点: {result.key_points}")   # List[str]: 关键要点列表
print(f"情感: {result.sentiment}")    # str: 情感倾向
print(f"字数: {result.word_count}")   # Optional[int]: 估计字数
print(f"类型验证: {type(result)}")    # <class 'ArticleAnalysis'>: Pydantic 对象类型
```

### 12.2 Token 机制详解

```python
# ============================================================
# Token 是大模型处理文本的最小单位，也是收费计量依据
# ------------------------------------------------------------
# 中文：1 Token ≈ 1~1.8 个汉字
# 英文：1 Token ≈ 3~4 个字母
# 计算工具：https://platform.openai.com/tokenizer
#
# 常见模型上下文窗口（影响可输入的最大 Token 数）：
#   gpt-4o          : 128k Token
#   gpt-4o-mini     : 128k Token
#   deepseek-v3     : 128k Token（DeepSeek 官方文档，Max Output 8k）
#   qwen-max        : 32k Token（通义千问，具体版本请以官网为准）
# ============================================================
from langchain_openai import ChatOpenAI  # ChatOpenAI: 对话模型封装

token_demo_model = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)  # ChatOpenAI: 低温稳定

# 调用后可通过两种方式获取 Token 使用量
response = token_demo_model.invoke("LangChain 是什么？用一句话回答。")  # AIMessage 对象

# ---- 方式 1（推荐）：usage_metadata — LangChain v1.x 统一标准字段 ----
# AIMessage.usage_metadata: Optional[UsageMetadata] — 跨 provider 的标准 Token 统计接口
# UsageMetadata 字段：
#   input_tokens  : int — 输入（提示词）消耗的 Token 数

usage = response.usage_metadata  # Optional[UsageMetadata]: 标准 Token 用量对象
if usage:                         # 不为 None 时打印（部分 Provider 可能不返回）
    print(f"输入 Token: {usage['input_tokens']}")   # int: 输入 Token 数
    print(f"输出 Token: {usage['output_tokens']}")  # int: 输出 Token 数
    print(f"总计 Token: {usage['total_tokens']}")   # int: 总 Token 数

# ---- 方式 2（兼容）：response_metadata — Provider 原始元数据（结构因 provider 而异）----
# 仅用于调试或需要 provider 特有字段时使用，字段名因供应商不同而不同
meta = response.response_metadata  # dict: OpenAI 原始响应元数据
print(f"原始元数据: {meta}")        # dict: 含 model_name、finish_reason 等字段
```

### 12.4 异步调用（ainvoke / astream / abatch）

```python
# ============================================================
# LangChain 所有 Runnable 均提供异步版本：
#   .ainvoke(input)     → 异步单次调用，返回 Coroutine[Any, Any, T]
#   .astream(input)     → 异步流式迭代，返回 AsyncIterator[chunk]
#   .abatch(inputs)     → 异步批量并行，返回 Coroutine[Any, Any, List[T]]
#
# 适用场景：
#   - FastAPI / Starlette 等异步 Web 框架
#   - 同时发起多个 LLM 请求（并发提速）
#   - Jupyter Notebook 中（直接 await，无需 asyncio.run）
# ============================================================
import asyncio  # asyncio: Python 异步 I/O 标准库
from langchain_openai import ChatOpenAI          # ChatOpenAI: 对话模型封装
from langchain_core.prompts import ChatPromptTemplate      # ChatPromptTemplate: 提示词模板
from langchain_core.output_parsers import StrOutputParser  # StrOutputParser: 字符串解析器

async_model  = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)   # ChatOpenAI: 模型实例
async_prompt = ChatPromptTemplate.from_messages([("human", "{input}")])  # 提示词模板
async_parser = StrOutputParser()                                    # StrOutputParser: 解析器
async_chain  = async_prompt | async_model | async_parser           # Runnable: 异步可调用链

async def demo_async():
    """
    展示三种异步调用方式，对比同步 invoke 效率。

    Returns:
        None — 直接打印各类异步调用结果
    """
    # ---- ainvoke：异步单次调用 ----
    # chain.ainvoke(input) -> Coroutine → str
    #   input  : dict — 输入变量字典
    #   返回值  : str（await 解包后）— 与 invoke 结果相同，但不阻塞事件循环
    result = await async_chain.ainvoke({"input": "用一句话解释递归"})  # str: 异步调用结果
    print(f"ainvoke 结果: {result}")

    # ---- astream：异步流式输出 ----
    # chain.astream(input) -> AsyncIterator[str]
    #   每次 yield 一个 str 片段（一个或多个 Token）
    print("astream 流式: ", end="")
    async for chunk in async_chain.astream({"input": "用一句话解释闭包"}):
        # chunk: str — 流式输出的单个文本片段
        print(chunk, end="", flush=True)  # 实时打印，不换行
    print()  # 流式结束换行

    # ---- abatch：异步批量并行（效率最高）----
    # chain.abatch(inputs) -> Coroutine → List[str]
    #   inputs  : List[dict] — 多个输入字典
    #   返回值  : List[str]（await 解包后）— 与输入一一对应，并发执行
    questions = [     # List[dict]: 批量问题输入
        {"input": "什么是装饰器？"},
        {"input": "什么是生成器？"},
        {"input": "什么是上下文管理器？"},
    ]
    results = await async_chain.abatch(questions)  # List[str]: 并发执行的批量结果
    for q, r in zip(questions, results):   # q: dict 输入，r: str 输出
        print(f"Q: {q['input']}\nA: {r}\n")  # 打印问答对

# Jupyter Notebook 中直接运行：await demo_async()
# 普通 Python 脚本中运行：
# asyncio.run(demo_async())
```

### 12.7 RunnablePassthrough.assign（多字段流水线）

```python
# ============================================================
# RunnablePassthrough.assign(**kwargs) — 在保留输入字典的基础上追加新字段
# 等价于旧版 SequentialChain 的多变量场景
# 适合多步流水线中「既保留原始输入，又追加每步中间结果」的场景
# ============================================================
from langchain_core.runnables import RunnablePassthrough  # RunnablePassthrough: 透传 Runnable
from langchain_core.prompts import ChatPromptTemplate     # ChatPromptTemplate: 对话模板
from langchain_core.output_parsers import StrOutputParser  # StrOutputParser: 字符串解析器
from langchain_openai import ChatOpenAI                   # ChatOpenAI: 模型实例

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)  # ChatOpenAI: 模型实例
parser = StrOutputParser()                                   # StrOutputParser: 解析器

# 第一步：产品名 → 定价（只输出整数）
price_prompt = ChatPromptTemplate.from_template(
    "只输出一个合理的整数价格（元），不要任何其他文字。产品：{product}"
    # {product}: str — 产品名称占位符
)

# 第二步：产品名 + 价格 → 促销文案
promo_prompt = ChatPromptTemplate.from_template(
    "为「{product}」（售价 {price} 元）写一句 40 字以内的促销文案。"
    # {product}: str — 原始产品名（来自初始输入，被 assign 保留）
    # {price}  : str — 上一步 LLM 生成的价格（被 assign 追加）
)

# RunnablePassthrough.assign(key=runnable) -> RunnableAssign
#   key     : str      — 追加到输入字典的新键名
#   runnable: Runnable — 用来计算该键值的子链
#   输入/输出: Dict → Dict（在原字典基础上新增 key 字段）
chain = (
    RunnablePassthrough.assign(
        price=price_prompt | llm | parser  # Runnable: 计算 price 字段，输出 str 价格
        # 此时输入 dict 从 {"product": "..."} → {"product": "...", "price": "299"}
    )
    | RunnablePassthrough.assign(
        promo_text=promo_prompt | llm | parser  # Runnable: 计算 promo_text 字段
        # 此时 dict → {"product": "...", "price": "299", "promo_text": "..."}
    )
)

result = chain.invoke({"product": "无线降噪耳机"})  # Dict: 最终包含所有字段的输出字典
# result: Dict[str, str] — 包含 product/price/promo_text 三个键
print(f"产品: {result['product']}")        # str: 原始产品名
print(f"定价: {result['price']} 元")       # str: LLM 生成的价格
print(f"文案: {result['promo_text']}")     # str: LLM 生成的促销文案
```

### 12.8 RunnableParallel（并行执行）

```python
# ============================================================
# RunnableParallel — 同一输入并行跑多条子链，结果合并为 dict
# 适合「翻译+摘要」、「多视角分析」、「多语言输出」等并行任务
# 内部自动并发执行各分支，性能优于顺序执行
# ============================================================
from langchain_core.runnables import RunnableParallel  # RunnableParallel: 显式并行 Runnable
from langchain_core.prompts import ChatPromptTemplate   # ChatPromptTemplate: 对话模板
from langchain_core.output_parsers import StrOutputParser  # StrOutputParser: 字符串解析器
from langchain_openai import ChatOpenAI                 # ChatOpenAI: 模型实例

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)  # ChatOpenAI: 模型实例
parser = StrOutputParser()                                   # StrOutputParser: 解析器

# 定义三条并行子链
translate_chain = (  # Runnable: 翻译链，输入 dict，输出 str（英文翻译）
    ChatPromptTemplate.from_template("把以下文本翻译成英文：{text}")
    | llm | parser
)

summarize_chain = (  # Runnable: 摘要链，输入 dict，输出 str（中文摘要）
    ChatPromptTemplate.from_template("用一句话总结以下文本的核心意思：{text}")
    | llm | parser
)

sentiment_chain = (  # Runnable: 情感分析链，输入 dict，输出 str（情感标签）
    ChatPromptTemplate.from_template("分析以下文本的情感倾向（正面/负面/中性），只输出一个词：{text}")
    | llm | parser
)

# RunnableParallel(key=runnable, ...) -> RunnableParallel
#   key     : str      — 结果字典中该子链输出对应的键名
#   runnable: Runnable — 并行执行的子链
#   输入     : dict — 所有子链共享同一份输入
#   返回值   : Dict[str, Any] — 各子链结果组成的字典
parallel = RunnableParallel(
    translation=translate_chain,  # Runnable: 翻译子链，结果存入 translation 键
    summary=summarize_chain,      # Runnable: 摘要子链，结果存入 summary 键
    sentiment=sentiment_chain,    # Runnable: 情感分析子链，结果存入 sentiment 键
)

text_input = {"text": "LangChain 是一个强大的 LLM 应用开发框架，让开发者能够快速构建 RAG 和 Agent 应用。"}

result = parallel.invoke(text_input)  # Dict[str, str]: 三条链并行执行的结果字典
print(f"英文翻译: {result['translation']}")  # str: 翻译结果
print(f"内容摘要: {result['summary']}")       # str: 摘要结果
print(f"情感倾向: {result['sentiment']}")     # str: 情感标签（正面/负面/中性）
```

### 12.9 RunnableBranch（条件路由）

```python
# ============================================================
# RunnableBranch — 根据条件将输入路由到不同子链
# 等价于旧版 RouterChain / MultiPromptChain 的现代写法
# 适合「按问题类型分流」、「多语言识别路由」等条件分支场景
# 更复杂的状态机/循环场景使用 LangGraph
# ============================================================
from langchain_core.runnables import RunnableBranch, RunnableLambda  # 分支 Runnable 组件
from langchain_core.prompts import ChatPromptTemplate     # ChatPromptTemplate: 对话模板
from langchain_core.output_parsers import StrOutputParser  # StrOutputParser: 字符串解析器
from langchain_openai import ChatOpenAI                   # ChatOpenAI: 模型实例

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)  # ChatOpenAI: 低温保证分类稳定
parser = StrOutputParser()                                   # StrOutputParser: 解析器

# 条件判断函数 — 判断输入属于哪种类型
def is_math_question(x: dict) -> bool:
    """
    判断输入是否为数学计算类问题。

    Args:
        x: dict — 包含 question 键的输入字典

    Returns:
        bool — True 表示数学问题，False 表示非数学问题
    """
    q = x.get("question", "")  # str: 获取问题文本
    # 检查是否含有数学相关关键词
    math_keywords = ["多少", "计算", "+", "-", "*", "/", "等于", "平方", "开根号"]
    return any(kw in q for kw in math_keywords)  # bool: 任意关键词命中则为数学问题

def is_code_question(x: dict) -> bool:
    """
    判断输入是否为编程相关问题。

    Args:
        x: dict — 包含 question 键的输入字典

    Returns:
        bool — True 表示编程问题，False 表示非编程问题
    """
    q = x.get("question", "")  # str: 获取问题文本
    code_keywords = ["代码", "编程", "函数", "bug", "报错", "Python", "Java", "API"]
    return any(kw in q for kw in code_keywords)  # bool: 任意关键词命中则为编程问题

# 各分支对应的子链
math_chain = (  # Runnable: 数学助手链
    ChatPromptTemplate.from_template("你是一位数学老师，请详细解题：{question}")
    | llm | parser
)
code_chain = (  # Runnable: 编程助手链
    ChatPromptTemplate.from_template("你是一位资深程序员，请提供代码解答：{question}")
    | llm | parser
)
general_chain = (  # Runnable: 通用助手链（默认分支）
    ChatPromptTemplate.from_template("你是一位博学的知识助手，请回答：{question}")
    | llm | parser
)

# RunnableBranch((condition, chain), ..., default_chain) -> RunnableBranch
#   condition    : Callable[[input], bool] — 条件判断函数，返回 True 时执行对应链
#   chain        : Runnable — 条件满足时执行的子链
#   default_chain: Runnable — 所有条件均不满足时执行的默认链（最后一项，不需要条件）
branch = RunnableBranch(
    (is_math_question, math_chain),   # (condition, chain): 数学问题 → math_chain
    (is_code_question, code_chain),   # (condition, chain): 编程问题 → code_chain
    general_chain,                    # Runnable: 默认链（无条件匹配时执行）
)

# 测试不同类型的问题路由
test_questions = [           # List[dict]: 覆盖三种分支的测试用例
    {"question": "12 * 34 + 5 等于多少？"},      # → 数学分支
    {"question": "Python 中如何处理 bug？"},      # → 编程分支
    {"question": "LangChain 是什么框架？"},       # → 通用分支（默认）
]

for q_dict in test_questions:        # q_dict: dict — 包含 question 键的输入字典
    print(f"\n问题：{q_dict['question']}")
    answer = branch.invoke(q_dict)   # str: 根据条件路由后执行对应子链的结果
    print(f"回答：{answer[:100]}...")  # str: 截取前 100 字展示
```

### 12.11 Callbacks（回调机制）

```python
# ============================================================
# Callbacks — LangChain 运行时事件钩子
# 允许在 LLM 调用的各个阶段插入自定义逻辑：
#   on_llm_start      : LLM 开始调用时触发
#   on_llm_end        : LLM 调用完成时触发（含最终输出）
#   on_llm_error      : LLM 调用报错时触发
#   on_chain_start    : Chain 开始时触发
#   on_chain_end      : Chain 完成时触发
#   on_tool_start     : Tool 开始调用时触发
#   on_tool_end       : Tool 调用完成时触发
# 常用于：日志记录、性能监控、Token 统计、调试分析
# ============================================================
from langchain_core.callbacks import BaseCallbackHandler  # BaseCallbackHandler: 回调基类
from langchain_openai import ChatOpenAI                   # ChatOpenAI: 对话模型
from langchain_core.prompts import ChatPromptTemplate      # ChatPromptTemplate: 提示词模板
from langchain_core.output_parsers import StrOutputParser  # StrOutputParser: 字符串解析器
from typing import Any, Dict, List, Union                  # 类型提示工具
import time  # time: 时间模块，用于性能计时

class TimingCallbackHandler(BaseCallbackHandler):
    """
    计时回调处理器：记录每次 LLM 调用的耗时和 Token 使用量。

    Attributes:
        start_time: float — LLM 调用开始的时间戳（Unix 时间）
    """

    def __init__(self):
        """初始化回调处理器，设置计时器起始值。"""
        super().__init__()
        self.start_time: float = 0.0  # float: LLM 调用开始时间戳

    def on_llm_start(
        self,
        serialized: Dict[str, Any],  # Dict: LLM 配置的序列化信息（模型名、参数等）
        prompts: List[str],           # List[str]: 发送给非对话 LLM 的提示词列表
        **kwargs: Any                 # Any: 其他关键字参数
    ) -> None:
        """非对话 LLM（OpenAI text-completion 等）开始调用时触发，记录开始时间。"""
        self.start_time = time.time()  # float: 记录调用开始时间戳
        print(f"[Callback] LLM 开始调用，发送 {len(prompts)} 条提示词")  # 打印回调触发信息

    def on_chat_model_start(
        self,
        serialized: Dict[str, Any],       # Dict: Chat Model 配置的序列化信息
        messages: List[List[Any]],         # List[List[BaseMessage]]: 发送给 Chat Model 的消息批次
        **kwargs: Any                      # Any: 其他关键字参数
    ) -> None:
        """Chat Model（ChatOpenAI 等）开始调用时触发，记录开始时间。
        
        注意：ChatOpenAI 是 Chat Model，触发 on_chat_model_start 而非 on_llm_start。
        两者均在 on_llm_end 结束，因此计时逻辑在此方法中启动。
        """
        self.start_time = time.time()  # float: 记录 Chat Model 调用开始时间戳
        total_msgs = sum(len(batch) for batch in messages)  # int: 消息总条数
        print(f"[Callback] Chat Model 开始调用，发送 {total_msgs} 条消息")  # 打印回调触发信息

    def on_llm_end(
        self,
        response: Any,  # LLMResult: LLM 调用的完整响应对象（含 generations 和 llm_output）
        **kwargs: Any   # Any: 其他关键字参数
    ) -> None:
        """LLM 调用完成时触发，打印耗时统计。"""
        elapsed = time.time() - self.start_time          # float: 调用耗时（秒）
        token_usage = getattr(response, "llm_output", {}) or {}
        usage = token_usage.get("token_usage", {})        # dict: Token 使用量字典
        print(f"[Callback] LLM 调用完成，耗时 {elapsed:.2f}s")     # 打印耗时
        if usage:                                          # Token 使用量不为空时打印
            print(f"[Callback] Token 消耗: 输入={usage.get('prompt_tokens', '?')}, "
                  f"输出={usage.get('completion_tokens', '?')}")

    def on_llm_error(
        self,
        error: Union[Exception, KeyboardInterrupt],  # Exception: 捕获到的异常对象
        **kwargs: Any                                 # Any: 其他关键字参数
    ) -> None:
        """LLM 调用报错时触发，打印错误信息。"""
        print(f"[Callback] LLM 调用失败: {error}")  # 打印异常信息

# 将回调处理器注入模型
timing_handler = TimingCallbackHandler()  # TimingCallbackHandler: 自定义回调处理器实例

cb_model = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0.0,
    callbacks=[timing_handler],  # List[BaseCallbackHandler]: 绑定回调处理器列表
)

cb_prompt = ChatPromptTemplate.from_messages([("human", "{input}")])  # 提示词模板
cb_chain   = cb_prompt | cb_model | StrOutputParser()  # Runnable: 带回调的完整链

result = cb_chain.invoke({"input": "用一句话解释 Callback 模式"})  # str: 调用结果
print(f"回答：{result}")  # str: 模型生成的回答
```

### 12.12 Legacy Chain 对照表（旧版写法迁移指南）

```python
# ============================================================
# LangChain v1.x 版本迁移说明
# ------------------------------------------------------------
# v0.3.x 中已废弃但可用的旧式 Chain，在 v1.x 中已完全从 langchain 包移除，
# 统一迁移至独立包 langchain-classic（需单独安装：pip install langchain-classic）。
# ============================================================

# ┌────────────────────────────────────────────────────────────────────────────────┐
# │  旧写法（v0.3.x 废弃 / v1.x 移至 langchain-classic）                          │
# │                              ↓ 现代推荐写法（LCEL / v1.x 原生）               │
# ├────────────────────────────────────────────────────────────────────────────────┤
# │ from langchain.chains import LLMChain           → prompt | llm               │
# │ LLMChain(llm, prompt, parser)                   → prompt | llm | parser      │
# │ .run() / .predict()                             → .invoke() / .batch()       │
# │ SimpleSequentialChain                           → chain1 | lambda | chain2   │
# │ SequentialChain（多变量）                       → RunnablePassthrough.assign  │
# │ RouterChain / MultiPromptChain                  → RunnableBranch              │
# │ ConversationChain + 旧 Memory                   → MessagesPlaceholder + 历史  │
# │ ConversationalRetrievalChain                    → LCEL RAG 链                 │
# │ LLMMathChain                                    → @tool 计算器 + Agent        │
# │ StuffDocumentsChain（旧类）                     → langchain_classic.chains…  │
# │ AgentExecutor / create_tool_calling_agent       → create_agent()              │
# │   （v1.x 中导入：from langchain_classic.agents import AgentExecutor）         │
# └────────────────────────────────────────────────────────────────────────────────┘

# ---- 旧写法示例（仅作对比，新项目禁止使用，v1.x 须改用 langchain-classic）----
# from langchain_classic.chains import LLMChain  # v1.x 旧式写法（需安装 langchain-classic）
# chain = LLMChain(llm=llm, prompt=prompt)
# result = chain.run("输入内容")  # ❌ .run() 已废弃，请用 .invoke()

# ---- 新写法示例（LCEL 推荐）----
from langchain_core.prompts import ChatPromptTemplate      # ChatPromptTemplate: 对话模板
from langchain_core.output_parsers import StrOutputParser  # StrOutputParser: 字符串解析器
from langchain_openai import ChatOpenAI                    # ChatOpenAI: 模型实例

llm    = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)  # ChatOpenAI: 模型实例
prompt = ChatPromptTemplate.from_messages([("human", "{input}")])  # 提示词模板
parser = StrOutputParser()  # StrOutputParser: 解析器

# ✅ LCEL 写法：prompt | llm | parser
modern_chain = prompt | llm | parser  # Runnable: 现代 LCEL 链

result = modern_chain.invoke({"input": "LCEL 相比旧版 LLMChain 有什么优势？"})
# str: LCEL 链执行结果
print(result)  # str: 模型对 LCEL 优势的解释

# 核心原则：「一切皆 Runnable，管道即架构」
# 所有组件（Prompt / Model / Parser / Retriever / Tool）均实现 Runnable 接口
# 用 | 串联即可构建任意复杂的 LLM 工作流，无需再学习各类专用 Chain 类
```

## 十三、总结

### 13.1 LangChain 核心架构回顾

```
langchain-core（核心抽象层）
├── Runnable 接口       → .invoke() / .stream() / .batch() / .ainvoke()
├── BaseMessage         → HumanMessage / AIMessage / SystemMessage
├── BasePromptTemplate  → ChatPromptTemplate / PromptTemplate
├── BaseOutputParser    → StrOutputParser / JsonOutputParser
└── BaseRetriever       → VectorStoreRetriever

langchain（高级封装层，v1.x）
├── LCEL 管道    → prompt | llm | parser（| 运算符串联 Runnable）
├── Agent        → create_agent（v1.x 推荐）
└── Memory       → RunnableWithMessageHistory（deprecated 1.3.3）/ ChatMessageHistory

langchain-classic（遗留兼容层）
├── Agent        → create_tool_calling_agent / AgentExecutor
└── Chain        → LLMChain / SequentialChain / ConversationalRetrievalChain 等

langchain-openai（OpenAI 集成）
├── ChatOpenAI          → 聊天模型接口（支持 Tool Calling / Structured Output）
└── OpenAIEmbeddings    → 文本向量化（text-embedding-3-small 推荐）

langchain-community（社区集成，500+ 种）
├── Document Loaders    → TextLoader / WebBaseLoader / PyPDFLoader 等
├── Vector Stores       → FAISS / Chroma / Milvus 等
└── Chat Histories      → ChatMessageHistory / RedisChatMessageHistory 等
```

### 13.2 学习路径建议

| 阶段 | 目标 | 核心知识点 |
|------|------|-----------|
| 入门 | 跑通第一个 LCEL 链 | `ChatOpenAI` + `ChatPromptTemplate` + `StrOutputParser` |
| 进阶 | 构建 RAG 应用 | 文档加载 + 分块 + 向量库 + `as_retriever()` + `RunnablePassthrough` |
| 高阶 | 实现多步 Agent | `@tool` + `create_agent`（v1.x）或 `create_tool_calling_agent`（langchain-classic） |
| 生产 | 可观测 + 持久化 | LangSmith 追踪 + Redis 持久化记忆 + 异步 `.ainvoke()` |
